Todo lo que necesitas para formular modelos. Lee esto y sigue la receta.

PASO 1 → Estructura de todo modelo

Todo problema de formulación tiene exactamente 5 partes, siempre en este orden:

1 Conjuntos e índices — ¿Sobre qué cosas decido?

2 Parámetros — ¿Qué datos me da el enunciado?

3 Variables de decisión — ¿Qué determino yo?

4 Función objetivo — ¿Qué maximizo o minimizo?

5 Restricciones — ¿Qué limitaciones existen?

Consejo: Lee TODO el enunciado antes de escribir. Subraya cada condición. Cada frase del enunciado se convierte en una restricción o en un parámetro.

PASO 2 → Variables: la decisión más importante

Pregúntate: ¿Qué tipo de decisión estoy tomando?

Tipo de decisiónVariableEjemplo
¿Cuánto? (cantidad) Continua: yᵢ ≥ 0 Monto a invertir, cajas a enviar
¿Sí o no? Binaria: xᵢ ∈ {0,1} ¿Selecciono proyecto i? ¿Uso medio m?
¿Cuántas unidades enteras? Entera: nᵢ ∈ ℤ⁺ Número de camiones, turnos
Patrón clave: Cuando hay un costo fijo (pagar por usar algo) + un costo variable (pagar por cada unidad), SIEMPRE necesitas dos variables vinculadas:
  • xᵢ ∈ {0,1} → "¿uso esta cosa?"
  • yᵢ ≥ 0 → "¿cuánto uso?"
  • Vínculo: yᵢ ≤ M · xᵢ (si no la uso, no puedo tener cantidad)

PASO 3 → Catálogo de restricciones

Restricciones de recurso / capacidad

"La bodega i tiene como máximo Rᵢ cajas"

Σⱼ xᵢⱼ ≤ Rᵢ ∀i ∈ I

Lo que produces/envías no puede superar lo que tienes.

Restricciones de demanda

"La ciudad j necesita al menos rⱼ cajas"

Σᵢ xᵢⱼ ≥ rⱼ ∀j ∈ J

Lo que recibe cada destino debe cubrir lo que necesita. Puede ser = si la demanda es exacta.

Presupuesto

"No gastar más de P"

Σⱼ cⱼ · xⱼ ≤ P

Límite de selecciones

"A lo más G proyectos" → Σ xⱼ ≤ G

"Al menos 1 de servicios básicos" → Σⱼ∈JSB xⱼ ≥ 1

Asignación exacta

"Cada curso va en exactamente un semestre"

Σₜ xᵢₜ = 1 ∀i ∈ I

Precedencias / requisitos

"Para tomar el curso i, debes haber aprobado los cursos en Pᵢ en semestres anteriores"

xᵢ₂ₜ₂ ≤ 1 - xᵢ₁ₜ₁ ∀i₁ ∈ I, i₂ ∈ Pᵢ₁, t₂ ≥ t₁

Esto dice: si el requisito i₂ está en semestre t₂, el curso i₁ no puede estar en t₁ ≤ t₂.

PASO 4 → Restricciones lógicas con binarias

Cuando el enunciado dice cosas como "si... entonces...", "o uno u otro", "no ambos", necesitas variables binarias y restricciones lógicas.

Frase del enunciadoRestricción
"i y j no pueden ir juntos" excluyentes xᵢ + xⱼ ≤ 1
"Si hago i, debo hacer k" requisito xᵢ ≤ xₖ
"Si hago 9, debo hacer 6 ó 7" requisito múltiple x₉ ≤ x₆ + x₇
"Si hago 1, NO puedo hacer 2 ni 3" x₁ ≤ 1 - x₂ y x₁ ≤ 1 - x₃
"Ambos juntos o ninguno" xᵢ = xₖ
"Si invierto en 4, a lo más el triple de lo de 10" y₄ ≤ 3·y₁₀
Estas restricciones son directas: no necesitan Big-M. Solo suman/restan binarias. Si puedes expresar la condición así, hazlo y no compliques con M.

🔥 PASO 5 → BIG-M (explicado de verdad)

¿Qué es M?

M es simplemente un número grande que usas como "interruptor" para prender o apagar una restricción.

Imagina que tienes una restricción que dice y ≤ 10. Quieres que a veces se cumpla y a veces no, dependiendo de una decisión. Lo que haces es:

y ≤ 10 + M · z

Donde z ∈ {0,1}. Piensa qué pasa:

  • Si z = 0: queda y ≤ 10 + 0 = 10 → la restricción ESTÁ ACTIVA, se cumple
  • Si z = 1: queda y ≤ 10 + M → si M es enorme (ej: 999999), esto es y ≤ 1000009 → la restricción es inútil, DESAPARECE
Eso es TODO lo que hace M. Es un número tan grande que cuando lo sumas, la restricción se vuelve irrelevante. Es un ON/OFF controlado por una binaria z.

¿Qué valor le pongo a M?

M debe ser lo suficientemente grande para que la restricción no moleste cuando está apagada, pero no pongas "M = infinito". Usa la cota natural del problema:

ContextoValor de MPor qué
Envío de ropa desde bodega i M = Rᵢ Nunca puedes enviar más de Rᵢ cajas
Inversión en proyecto n M = Mₙ El máximo que puedes invertir es Mₙ
Apariciones de producto en medio M = MaxApₘₜ El medio no acepta más que eso
Número de bodegas que sirven a j M = n - 1 A lo más n bodegas, el exceso sobre 1 es n-1
Créditos de cursos M = 1 + max Cᵢ La mayor diferencia posible entre créditos
Truco: Pregúntate "¿cuál es el valor MÁXIMO que podría tomar el lado izquierdo de mi restricción?" — eso es tu M.

Situación 1: "O se cumple A, o se cumple B"

Ejemplo: "O el curso i tiene más créditos que j, o j tiene más que i" (no pueden tener los mismos)

Tienes dos restricciones, y al menos una debe cumplirse:

f(x) ≤ 0 ...restricción A g(x) ≤ 0 ...restricción B

Introduces z ∈ {0,1} y escribes:

f(x) ≤ M · z ← si z=0, A se cumple; si z=1, A se apaga g(x) ≤ M · (1 - z) ← si z=1, B se cumple; si z=0, B se apaga

Así siempre una de las dos está activa.

Ejemplo real (Control 2024, parte b): En semestre 4, no pueden haber dos cursos con los mismos créditos.

Condición: si xi4 = 1 y xj4 = 1, entonces Cᵢ ≠ Cⱼ.
Cᵢ ≠ Cⱼ significa: o Cᵢ - Cⱼ ≥ 1, o Cⱼ - Cᵢ ≥ 1.
Reescrito como ≤ 0: o (1 - Cᵢ·xi4 + Cⱼ·xj4 ≤ 0), o (1 - Cⱼ·xj4 + Cᵢ·xi4 ≤ 0).

1 - Cᵢ·xi4 + Cⱼ·xj4 ≤ M·wᵢⱼ
1 - Cⱼ·xj4 + Cᵢ·xi4 ≤ M·(1 - wᵢⱼ)

Situación 2: "Si pasa algo, entonces debe cumplirse otra cosa"

Esta es la MÁS COMÚN en controles.

Estructura: "Si f(x) > 0, entonces g(x) ≤ 0"

El truco: esto es EQUIVALENTE a decir "O f(x) ≤ 0, O g(x) ≤ 0". Piénsalo: si f(x) > 0 (la condición se cumple), entonces la única opción que queda es que g(x) ≤ 0. Si f(x) ≤ 0 (la condición no se cumple), no importa g.

Para variables enteras, f(x) > 0 es lo mismo que f(x) ≥ 1, o sea f(x) - 1 ≥ 0:

f(x) - 1 ≤ M · z g(x) ≤ M · (1 - z) z ∈ {0,1}
Ejemplo real (Control 2025, parte d): "Solo una entre bodegas i e i' puede enviar ropa."

Condición: si envío ropa desde i (Σⱼ xᵢⱼ > 0), entonces no envío desde i' (Σⱼ xᵢ'ⱼ = 0).

f(x) = Σⱼ xᵢⱼ (si esto > 0, se activa la condición)
g(x) = Σⱼ xᵢ'ⱼ (esto debe ser ≤ 0)

Σⱼ xᵢⱼ ≤ M · w con M = Rᵢ
Σⱼ xᵢ'ⱼ ≤ M · (1-w) con M = Rᵢ'
w ∈ {0,1}
Ejemplo real (Control 2025, parte e): "Penalización fⱼ si ciudad j recibe de más de una bodega."

Paso 1: crear zᵢⱼ ∈ {0,1} = 1 si envío algo de i a j:
xᵢⱼ + yᵢⱼ ≤ (Rᵢ + Aᵢ) · zᵢⱼ

Paso 2: contar bodegas que sirven a j: Σᵢ zᵢⱼ

Paso 3: si Σᵢ zᵢⱼ > 1, cobrar penalización. Esto es "si f(x) > 0, entonces vⱼ = 1":
f(x) = Σᵢ zᵢⱼ - 1
Σᵢ zᵢⱼ - 1 ≤ M · vⱼ con M = n - 1

Paso 4: sumar penalización a la F.O.: + Σⱼ fⱼ · vⱼ

Situación 3: Activar variable binaria con variable continua

"Si no selecciono el proyecto, no puedo invertir en él"

yₙ ≤ Mₙ · xₙ ∀n

Si xₙ = 0 → yₙ ≤ 0 → no inviertes nada.
Si xₙ = 1 → yₙ ≤ Mₙ → puedes invertir hasta el máximo.

Esto NO es Big-M complicado. Es el uso más simple: vincular una continua con su binaria de activación.

Resumen visual de Big-M

┌─────────────────────────────────────────────────┐ │ ¿Qué necesito? │ Qué escribo │ ├─────────────────────────────────────────────────┤ │ Vincular continua y │ y ≤ M · x │ │ su binaria │ │ ├─────────────────────────────────────────────────┤ │ "O A o B" │ f(x) ≤ M·z │ │ │ g(x) ≤ M·(1-z) │ ├─────────────────────────────────────────────────┤ │ "Si A entonces B" │ f(x)-1 ≤ M·z │ │ (con enteros) │ g(x) ≤ M·(1-z) │ ├─────────────────────────────────────────────────┤ │ Penalización/costo fijo │ Vincular + sumar │ │ condicional │ costo·v a la F.O. │ └─────────────────────────────────────────────────┘

PASO 6 → Función objetivo: patrones comunes

TipoF.O.
Maximizar utilidadmax Σ (ingreso - costo_variable)·y - Σ costo_fijo·x
Minimizar costo de transportemin Σᵢ Σⱼ cᵢⱼ · xᵢⱼ
Maximizar beneficio socialmax Σⱼ bⱼ · xⱼ
Minimizar costo + penalizaciónmin Σ cᵢⱼ·xᵢⱼ + Σ fⱼ·vⱼ
Minimizar el máximo (holgura, ciclo)min H s.a. zₜ ≤ H ∀t
"Minimizar el máximo" es un patrón especial: creas variable H, obligas a que H ≥ cada valor individual con zₜ ≤ H, y minimizas H. Apareció en el control 2024 (holgura de malla) y en línea de ensamblaje.

PASO 7 → Checklist final antes de entregar

Antes de soltar el lápiz, revisa:

  • ☐ ¿Definí todos los conjuntos e índices?
  • ☐ ¿Cada parámetro del enunciado tiene nombre?
  • ☐ ¿Las variables tienen dominio declarado? (xᵢ ∈ {0,1}, yᵢ ≥ 0)
  • ☐ ¿La F.O. dice min o max?
  • ☐ ¿Cada restricción tiene ∀i, ∀t, ∀j donde corresponde?
  • ☐ ¿Cubrí TODAS las condiciones del enunciado? (relee y cuenta)
  • ☐ ¿Las unidades son consistentes? (no sumar pesos con cajas)
  • ☐ ¿Cada M tiene un valor justificado, no solo "M grande"?

Recuerda: cada frase del enunciado = una restricción.
