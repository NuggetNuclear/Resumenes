¿Qué es un SD?

Coulouris: componentes en red que se coordinan solo por paso de mensajes.

Tanenbaum: colección de máquinas independientes que aparecen como sistema único y coherente.

SD ≠ Red: en SD el sistema invoca la acción (hay middleware). En red, el usuario interactúa con cada máquina.

3 técnicas: Particionar · Replicar · Cache

Sharding (partición horizontal)

  1. Hash-Based: hash sobre atributo. Balance uniforme.
  2. Range: por rangos ordenados (fechas, IDs).
  3. Geographical: por región (latencia, regulación).
  4. Directory/Virtual Bucket: lookup centralizado.

Contras: hot-spots, complejidad, resharding caro.

Ojo V/F: las particiones NO necesitan tener recursos similares — depende de la distribución de consultas.

Replicación

Pros: fault-tolerance · accesos paralelos · balance de carga.

Contras: costo + mantener estado (depende de N réplicas y ratio read/write).

Red: Performance

Tm = tL + b / tT
  • tL (latencia): msg vacío. Sw overhead, ruteo, carga.
  • tT (BW): bit/s. Características físicas.
  • Jitter: variación de tiempo. Clave en multimedia.

Redes: LAN (alta vel) · MAN (regional) · WAN (lenta, distante) · Wireless · Internetworks.

Stack de Comunicación

App ↓ Middleware (RPC, mensajes) ↓ OS Comms (sockets, TCP/UDP) ↓ Network (bits).

Protocolo: reglas de codificación, inicio/fin, detección de errores, representación (endianness).

3 Paradigmas de Comunicación

  1. IPC (Interprocess): directa, bajo nivel. Sockets, MPI.
  2. Remote Invocation: request/reply bidireccional. RPC, RMI.
  3. Indirect: tercera parte. Space + time uncoupling. Pub/Sub, MQ.

RPC

Invocar procedimiento remoto sin conocer la red. Modelo C/S, síncrono.

Stubs

  • Cliente: marshalling → envía → recibe → unmarshalling.
  • Servidor: recibe → unmarshalling → llama local → retorna.

Marshalling: transformar datos a formato de red (serialización).

Parámetros

Por valor (endianness, ASCII/EBCDIC) · Por referencia (copiar array completo).

Async

Servidor envía ACK inmediato → cliente no se bloquea (multi-thread).

V/F trampa: RPC NO es asíncrono por defecto. Es síncrono, C/S, por paso de mensajes.

RMI (Java)

RPC orientado a objetos (OOM). Stub (cliente) + Skeleton (servidor). Enmascara objeto remoto con proxy.

Arquitecturas

Centralizada (C/S): request-reply, escalamiento vertical, cuellos de botella, seguro, recursos dedicados.

Descentralizada (P2P): simétrica, directa, escalabilidad alta.

Híbrida (Edge): procesamiento cerca del usuario. Akamai, Open Connect.

C/S: 2-tier vs 3-tier

3-tier: servidor puede actuar como cliente de otro. Thin client > fat client (más mantenible).

P2P Estructurado

  • Determinístico · DHT (SHA-1/MD5)
  • KBR (Key-Based Routing) · O(log N)
  • Escala a 100M+ · búsquedas puntuales

Contras: alta latencia, no seguro, solo búsqueda exacta, churn.

Chord

Anillo 0..2^m · ID=SHA(IP+puerto) · asigna al sucesor · finger table = log m peers → salta peers.

Pastry

128 bits · ruteo por prefijo (Plaxon-Tree) · 3 tablas: leafset + routing table + neighbours.

Peer cae

Solo se afectan sus pares; sucesor toma responsabilidad. Réplicas en S sucesores.

P2P No Estructurado

Aleatorio · conocimiento local · sin garantía de encontrar.

Búsquedas

  • Flooding: a todos. No escala.
  • BFS: flooding + TTL. Muchos duplicados.
  • Iterative Deepening: BFS secuenciales, profundidad creciente. Caché intermedio.
  • Directed BFS: subset de vecinos elegido.
  • Random Walk: 1 vecino random. Ciego, baja msg, alta latencia.
  • K-Random Walks: paralelo. APS, Id-based, Two-Level.

P2P Híbrido — Super-Peers

Peers con más capacidad = índices/brokers. Jerarquía. Peers regulares → SP. Se pierde simetría. Ej: Skype.
